## 1. プロジェクト概要

### 1.1 プロジェクト名
PictureStory（仮）

### 1.2 ジャンル
HD-2D横スクロールアクションRPG

### 1.3 コンセプト
2Dドット絵キャラクターと3D背景を融合したHD-2Dスタイルで、探索・戦闘・RPG要素を持つ横スクロールアクションゲーム。

### 1.4 世界観
**「絵の世界版トイストーリー」** - キャラクターはカード、武器はフィルム、アタッチメントはシールとして表現される、絵画の世界を舞台にした物語。メイプルストーリーのような世界観でモンスターを仲間にできる。

### 1.5 コアゲームプレイ
**「アイテムを獲得する楽しさ」** を中心とした以下の体験：
1. 敵がアイテムを沢山ドロップする
2. 敵がステータスそのまま仲間になる（レベルは1からだが、元のレベルまで上げると敵対時のステータスになる）
3. ごく稀に貴重な装備をドロップする
4. レベルアップでめちゃくちゃ強くなる
5. アビリティツリー次第で多様なプレイができる
6. プレイヤー同士のコミュニケーション（将来機能）
7. 協力プレイでより強い敵を倒せるようになる（将来機能）

### 1.6 規模
インディーゲーム（商用リリースを目指す中規模プロジェクト）

### 1.7 ゲーム形式
- **オンラインゲーム**: サーバー接続必須のリアルタイムオンラインゲーム
- **リアルタイム操作**: インベントリやメニュー操作中もゲームは停止しない（ポーズ機能なし）
- **マルチプレイ対応**: 将来的に協力プレイ・PvP機能を実装予定（セクション12参照）

## 2. ターゲット環境

### 2.1 プラットフォーム
- **メインターゲット**: Mobile (iOS/Android)
- **最小スペック（想定）**:
  - iOS: iOS 13.0以降、iPhone 8以降
  - Android: Android 8.0以降、RAM 3GB以上

### 2.2 操作方法
- タッチスクリーン操作
  - バーチャルパッド（移動）
  - タッチボタン（ジャンプ、攻撃、アイテムスキル）
  - スワイプジェスチャー（特殊アクション）

### 2.3 画面方向
- 横向き（ランドスケープモード）

### 2.4 操作体系
シンプルな5操作で全てのアクションを実行可能：

| 操作 | アクション | 補足 |
|------|----------|------|
| 画面タッチ | 攻撃 | 選択中アイテムスキルを発動 |
| 左右操作 | 移動 | バーチャルパッドまたはスワイプ |
| 上操作 | ジャンプ | 長押しで高ジャンプ |
| 下操作 | ガード | 長押しで継続ガード |
| スロットタッチ（10枠） | アイテム/アイテムスキル使用 | 各スロットに装備をセット |

**設計方針**:
- 片手操作でも基本プレイが可能
- 補助機能（オートエイム等）との組み合わせでカジュアルプレイをサポート
- 両手操作による精密なプレイも可能

## 3. 技術スタック

### 3.1 ゲームエンジン
- **Unity** (推奨バージョン: 2022 LTS以降)

### 3.2 主要な技術・アセット
- **描画**: Universal Render Pipeline (URP)
- **2Dシステム**: Unity 2D Sprite/Tilemap
- **3D背景**: Unity 3Dモデリング + Post-Processing
- **物理エンジン**: Unity Physics 2D
- **アニメーション**: Unity Animator + 2D Animation
- **エフェクト**: Particle System / Shader Graph
- **セーブデータ**: JSON / PlayerPrefs または SQLite
- **多言語対応**: Localization Package

### 3.3 アートアセット
- **キャラクター**: 2Dドット絵スプライト（32x32 or 64x64基準）
- **背景**: 3Dモデル + ライティング + 被写界深度
- **UI**: HD解像度対応（1920x1080基準）

## 4. HD-2Dビジュアルスタイル定義

### 4.1 HD-2Dとは
2Dドット絵キャラクターと3D背景を組み合わせ、ライティングや被写界深度などの最新技術を用いて立体感と奥行きを表現する視覚スタイル。

### 4.2 実装要件
- **2Dキャラクター**:
  - ドット絵スプライト
  - 8方向または多方向のアニメーション
  - エッジライティング対応

- **3D背景**:
  - 奥行きのある3Dモデル
  - リアルタイムライティング（動的な光源変化）
  - パーティクルエフェクト（雨、雪、霧など）

- **カメラワーク**:
  - 横スクロールを基本としつつ、重要なシーンでカメラアングル変更
  - パララックススクロール（多層背景）
  - 被写界深度（ボケ効果）

- **ポストプロセス**:
  - Bloom（光の滲み）
  - Color Grading（色調補正）
  - Vignette（周辺減光）

## 5. ゲームプレイメカニクス

### 5.1 基本アクション

> **重要**: プレイヤーが操作するキャラクターは固定の主人公ではなく、ゲームプレイを通じて収集した保有キャラクターの中からプレイヤーが選択する。詳細は「5.5 キャラクター収集・選択システム」を参照。

- **移動**: 左右移動、ダッシュ
- **ジャンプ**: 通常ジャンプ、二段ジャンプ（AP/アイテムで解放）
- **攻撃**:
  - 基本攻撃（3連コンボ）
  - チャージ攻撃
  - 空中攻撃
- **空中アクション**:
  - 空中方向転換（初期から可能、AP解放で操作性向上）
  - 空中アイテムスキル使用（初期から可能、AP解放で使用可能アイテムスキル拡張）
  - 敵を踏んでジャンプリセット（AP/アイテムで解放）
  - 空中ダッシュ（アイテムで解放）
  - 壁キック（アイテムで解放）

### 5.2 戦闘システム

#### 5.2.1 武器分類
武器は以下の4種類に分類され、それぞれ特性が異なる：

| 武器種 | 射程 | 特性 |
|--------|------|------|
| **打撃** | 近距離 | 肉体に追加ダメージ（アーマーには通常ダメージ） |
| **斬撃** | 近距離 | アーマー貫通ダメージ |
| **銃撃** | 遠距離 | 弾を消費してアーマー貫通ダメージ。弾がなくても属性弾使用可能 |
| **爆撃** | 遠距離 | 爆弾を消費して肉体に追加ダメージ（アーマーには通常ダメージ） |

#### 5.2.2 武器ステータス
- 攻撃倍率（キャラクターの攻撃力と掛け合わせ）
- 属性倍率（キャラクターの属性値と掛け合わせ）
- 攻撃レート（アイテムスキルのクールダウン）
- 命中率
- 射程距離（この距離を超えるとダメージ0）
- 操作性（攻撃が出るまでの時間に影響）
- 重さ（スタミナ使用量）
- 装弾数（遠距離武器のみ）
- 耐久値（ゼロになるとステータスが低下）

#### 5.2.3 アイテムスキルシステム
- 4枠の攻撃スロットにアイテムスキルをセット可能
- クールダウン経過後に使用可能
- アイテムスキル発動タイミング：タッチ（青）、ホールド（緑）、リムーブ（黄色）
- 選択中のアイテムスキルは枠が光る

#### 5.2.4 回避/防御
- **ガード**: スタミナを消費してキャラクターの向いている前方に行える
- **ローリング回避**:
  - 無敵時間＋クールダウンあり（Dead Cells型）
  - **アイテムの能力として実装**（初期状態では使用不可）
  - 高レアリティアイテムで回避性能向上（無敵時間延長、クールダウン短縮）
- **パリィ**:
  - タイミング重視の防御アクション
  - 成功時: **スタミナ回復＋反撃チャンス**
  - 一定レアリティ以上の能力で攻撃反射可能

### 5.3 敵キャラクター
- **雑魚敵**: パターン行動、各エリアに特有の敵
- **ミニボス**: 中ボス、特殊な攻撃パターン
- **ボス**: 各章のボス、複数フェーズの戦闘

### 5.4 状態異常システム
**アイテムスキル**により以下の状態異常を付与できる（属性による付与ではない）：

| 状態異常 | 効果 |
|---------|------|
| **昏睡** | 行動不能、ガード不可 |
| **憂鬱** | アイテム/アイテムスキル使用までの時間が遅延 |
| **出血** | 継続ダメージ |
| **不安** | スタミナが強制ゼロ |

- 状態異常はアイテムでデバフを緩和・解消可能
- 他の状態異常にかかりやすくなったりダメージを受けたりデメリットもある
- Lv100以降のステージで状態異常付与アイテムスキルを持つ敵・装備が登場

### 5.5 キャラクター収集・選択システム

#### 基本概念
**このゲームに固定の主人公キャラクターは存在しない。** プレイヤーはゲームプレイを通じてキャラクターを収集し、保有コレクションの中から「操作キャラクター」を自由に選択してプレイする。

| 役割 | 定義 |
|------|------|
| **操作キャラクター** | プレイヤーが直接操作する1体。保有コレクションから選択 |
| **パートナーキャラクター** | AIが自動操作する最大2体。保有コレクションから選択 |
| **保有コレクション** | 収集済みキャラクターの一覧（基本上限50体） |

#### キャラクター入手方法
- **ボス撃破ドロップ**: ボスを倒した時のみキャラクタードロップが発生（雑魚敵からはドロップしない）
- **ドロップ率**: オーバーキル具合に応じて変動
  - HP0ちょうどで倒す: 50% × キャラ固有確率
  - HPの倍でオーバーキル: 10% × キャラ固有確率
- **ストーリー開始時**: チュートリアルでたまたま落ちているキャラクターアイテムを拾って最初の操作キャラクターを入手（ゲーム開始直後は保有キャラクターが0体のため）
- **ガチャ**: 「集会所」ガチャで新キャラクターを入手（10.4参照）

#### 出撃画面（ステージ開始前）
ステージに入る前の出撃画面で以下を設定する：

1. **操作キャラクター選択**: 保有コレクションから1体を選択 → プレイヤーが直接操作
2. **パートナー選択**: 保有コレクションから最大2体を選択 → AIが自動操作
   - 小型キャラクターは2体で1体分のスロットを使用
3. **装備設定**: 各キャラクターに武器・防具・アクセサリーをセット
4. **難易度選択**: ステージ難易度の設定（ステージ選択時のみ変更可）

#### 保有コレクション管理
- **登録上限**: 基本50体まで（課金で50体単位追加）
- **同一キャラ入手時**: 既存キャラの基礎ステータスアップ(+2〜5%) or 追加登録を選択
- **データ構造**: 操作キャラクターとパートナーは同一のキャラクターデータ構造を使用
- **ステータス継承**: 敵として登場したレベルまで育てると、敵対時と同等のステータスに到達

#### パートナー制御
- **装備共有**: パートナーもプレイヤーと装備を兼用可能（1アイテム＝1体のみ装備可、出撃画面でセット変更）
- **AI制御**: 初期は指示型（攻撃/防御/待機コマンドで制御）、将来的に戦術型へ移行予定

#### キャラクターレアリティ
- キャラクターはアイテムと同一のレアリティ体系（コモン〜セイクリッド 21段階）を持つ
- レアリティがレベルキャップを決定する（詳細は `Status - Rarelity.csv` 参照）
- レベルキャップ到達によって新たな能力が解放される
- 敵キャラクターのレアリティはステージと難易度によって固定

#### ステージ中のキャラクター切り替え
- 画面上に常時表示されているパーティUI（名前＋ステータス）をタップで操作キャラクターを切り替え
- 切り替え後、元の操作キャラクターはAI操作に戻る
- コスト・クールダウンなし
- 死亡済みキャラのUIは表示されるが選択不可
- 操作キャラクター死亡時は自動的に次の生存キャラクターへ引き継ぎ（全員死亡でゲームオーバー）

#### 死亡・リスポーン

| 状況 | 挙動 |
|------|------|
| 操作キャラ死亡 | 自動で次の生存キャラに操作移譲 |
| 全員死亡（オンライン） | ゲームオーバー → 課金リスポーン（全員全回復）or 拠点送還（難易度別死亡ペナルティ） |
| 全員死亡（オフライン） | 最後のチェックポイントから再開。全員HP1/4・その他初期状態で復活 |

#### コレクション管理メニュー（拠点でいつでも実施可能）

| 操作 | 内容 | 返還物 |
|------|------|--------|
| **おわかれ** | キャラクターを解放。キャラクター種別ごとに固定されたアイテム候補からランダムドロップ | キャラ固有アイテム |
| **スクラップ** | キャラクターを解体してレベルコストの半額を回収 | 通貨（レベルコスト × 0.5） |
| **錬金術** | 同一キャラクターを素材に合成してレアリティを1段階上昇させる | 素材の装備品・通貨（レベルコスト × 0.5） |

**錬金術の条件**:
- 素材は「同一キャラクター種別」かつ「ベースキャラ以上のレアリティ」のものだけ選択可能
- 必要素材数はベースのレアリティによって異なる
- 一部の特別なキャラクターは錬金術不可（データフラグで管理）

**コレクション満員時の超過取得**:
- ステージ上で超過取得したキャラクターは拠点帰還まで「保留」
- 拠点に戻ると上記3種のいずれかで1体を削除するよう強制される（保留キャラまたは既存キャラのどちらかを削除）
- 保留中にゲームオーバーになった場合、保留キャラは消滅

### 5.6 RPG要素
- **レベルシステム**:
  - **通貨消費によるレベルアップ**（戦闘で経験値は獲得しない）
  - レベルアップに必要な通貨量はレベルに応じて増加
  - HP/MP/攻撃力/防御力の成長
  - パートナーも同様に通貨でレベルアップ

- **アビリティツリー**:
  - 複数の成長ルート
  - パッシブアビリティ、アクティブアビリティ

- **装備システム**:
  - 武器、防具、アクセサリー
  - 装備による能力値変化
  - レアリティシステム: 21段階（コモン～セイクリッド）
    - 詳細は`game_system_design.md`のセクション8.1を参照
  - **入手方法**:
    - 敵からのドロップ（通常装備）
    - NPCショップでの購入（通常装備）
    - クラフト/素材合成（通常装備）
    - 限定装備ガチャ（期間限定装備のみ）

- **アイテムシステム**:
  - 回復アイテム
  - バフアイテム
  - 素材アイテム（クラフト用）

### 5.7 探索要素
- **マップ探索**:
  - メトロイドヴァニア風の広大なマップ
  - 能力解放による行動範囲拡大
  - 隠し通路、隠しアイテム

- **謎解き**:
  - スイッチパズル
  - 移動パズル（ブロック移動など）
  - 時限要素のあるギミック

- **収集要素**:
  - コレクタブルアイテム
  - 実績/トロフィー
  - 図鑑（敵、アイテム）

### 5.8 コンボシステム
連続でオーバーキルを達成することで報酬が増加するシステム：

- **コンボカウント**: オーバーキル達成で+1、一定時間経過でリセット
- **ドロップボーナス**:
  - コンボ数に応じて通貨ドロップ量が増加
  - コンボ数に応じてアイテムドロップ率が増加
- **キャラクタードロップ**: ボスのみ（雑魚敵からはドロップしない）
- **設計思想**: 効率的なプレイを報酬で促進、スキルプレイヤーに有利

### 5.9 シナジーシステム
アイテムスキルと装備の組み合わせによる相乗効果：

- **アイテムスキルシナジー**:
  - 特定アイテムスキルの組み合わせでボーナス効果
  - 属性連携（例：油アイテムスキル→火アイテムスキルで追加ダメージ）
  - Hades/Slay the Spire的な「ビルド構築」の楽しさを重視
- **装備セット効果**:
  - 同シリーズ装備を揃えることでセットボーナス
  - 完全セットでプレイスタイルを大きく変化させる
  - セットごとに特化ビルドを定義
- **設計方針**: 他ゲーム以上のシナジー数を実装目標

### 5.10 攻略自由度
- **シーケンスブレイク**: 複数ルートを公式にサポート、RTA向け設計
- **ボス攻略多様性**:
  - ボス/マップごとの固有勝ち筋
  - キャラクター属性/スキルによる固有勝ち筋
  - 環境ギミック、地形利用、パートナー戦術など多彩なアプローチ

### 5.11 環境インタラクション
属性と環境の連動：

- **基本インタラクション**（初期実装）:
  - 水場での雷属性拡散
  - 油場への火属性引火
  - 草の燃焼
- **物理演算**（将来実装）:
  - 爆発での地形変化
  - 液体の流動
  - 連鎖反応

### 5.12 難易度システム
ストーリー進行に応じた難易度変化：

| ステージ難易度 | 解放条件 | 死亡ペナルティ | 対象プレイヤー |
|--------------|---------|--------------|--------------|
| ノーマル | 初期解放 | 通貨の一部ロスト | カジュアル層 |
| ハード | Lv100以降・ストーリー進行 | アイテムロスト（保護可能） | コア層 |

**設計方針**:
- 難易度選択ではなくストーリー進行で自然に難易度が上昇
- カジュアル層はノーマル難易度で十分クリア可能
- コア層はハード難易度で緊張感のあるプレイを楽しめる
- ペナルティが重くなる代わりに報酬も増加

### 5.13 段階解放システム
ゲーム進行に応じてシステムが段階的に解放：

#### 初期状態（〜Lv100）
| 要素 | 初期解放 | 解放数 |
|-----|---------|--------|
| 属性 | 火、水、木、雷、土 | 5種類 |
| レアリティ | コモン〜ウルトラレア＋ | 6段階 |
| 状態異常 | なし | 0種類 |

#### Lv100以降（ストーリー進行で順次解放）
| 要素 | 追加解放 | 解放後合計 |
|-----|---------|-----------|
| 属性 | 氷、油、毒、風、魂 | 10種類 |
| レアリティ | ハイパーレア〜セイクリッド | 21段階 |
| 状態異常 | 昏睡、憂鬱、出血、不安（アイテムスキルで付与） | 4種類 |

**属性解放クエスト**:
- 各属性に専用の解放クエストが存在
- クエストは任意の順番でクリア可能
- 解放することで新属性のキャラクター・装備が使用可能に

**設計方針**:
- 初期は少ない要素で基本を学習
- 進行に応じて選択肢が増加し、戦略の幅が広がる
- 情報過多による離脱を防止

## 6. ゲーム構成

### 6.1 ストーリー構成
- **章立て**: 5-8章構成
- **プレイ時間**: 10-15時間（メインストーリー）
- **サイドクエスト**: オプショナル要素として追加

#### ストーリー概要
1. **チュートリアル**: 100レベルのキャラクターを操作中に味方拠点（アートワールド）がNPCに襲われる
2. **序盤**: 敗北後、たまたま落ちていたキャラクターアイテムを拾って操作を開始
3. **中盤**: 元々拠点に居たキャラクターを取り戻す旅
4. **100レベル付近**: 拠点を襲ったキャラクターと再度相対（実はラスボスに操られていた）
5. **100レベル以降**: 氷、油、毒、風、魂属性のステータスが解放クエストで順次開放
   - 各属性に専用の解放クエストが存在
   - クエストは任意の順番でクリア可能
6. **終盤**: ボスを仲間にすると拠点が大きく開放され、全キャラクターが収まるスペースが解放
7. **ラスボス**: 「Administrator」という名前、全ステータスがカンストしたキャラクターを使用（倒さなくても良い）

### 6.2 マップ構成
- **拠点（村/町）**:
  - NPC会話
  - ショップ
  - クエスト受注

- **フィールド**:
  - 森、洞窟、城、塔など多様な環境
  - 各エリア10-15分の探索時間

- **ダンジョン**:
  - 謎解き要素が多い
  - 最後にボス戦

### 6.3 セーブシステム
- **オートセーブ**: チェックポイント通過時
- **マニュアルセーブ**: 拠点やセーブポイントで
- **セーブスロット**: 3つまで

### 6.4 死亡ペナルティ
プレイヤーが死亡した場合、以下のペナルティが発生する：
- **アイテムロスト**: 所持しているアイテムを失う（**通貨は除く**）
- **リスポーン**: 最後のチェックポイントから再開
- **キャラクター**: 登録済みキャラクターは失わない

#### アイテム保護システム
- **出撃前保護**: 出撃画面でアイテムにレアリティに応じた通貨を支払うことで、死亡時のロストから保護可能
- **保護コスト**: レアリティが高いほど保護に必要な通貨が増加
- **設計思想**: 緊張感のあるプレイ＋保護による救済策の両立

※課金アイテムで即リスポーンや切断時無敵状態などの保護機能も利用可能
