## 4. 戦闘システム

### 4.1 ダメージ計算フロー

既存実装: `DamageManager.cs`

```
攻撃発生
  ↓
┌─────────────────────────────────┐
│ 1. 基礎ダメージ計算              │
│    CalcAttack()                 │
│    - アイテムスキルパワー × 0.5         │
│    - 運によるランダム変動        │
│    - クリティカル判定            │
└─────────────────────────────────┘
  ↓
┌─────────────────────────────────┐
│ 2. 防御計算                      │
│    CalcDefence()                │
│    - 防御力で減算（× 0.25）     │
│    - 属性相性計算                │
└─────────────────────────────────┘
  ↓
┌─────────────────────────────────┐
│ 3. シールド計算                  │
│    CalcShield()                 │
│    - シールドで吸収              │
│    - シールドダメージ適用        │
└─────────────────────────────────┘
  ↓
┌─────────────────────────────────┐
│ 4. 最終ダメージ適用              │
│    - HP減少                     │
│    - ダメージモーション          │
│    - ダメージ数値表示            │
└─────────────────────────────────┘
```

### 4.2 ダメージ計算式詳細

#### 4.2.1 基礎ダメージ計算

```
基礎ダメージ = アイテムスキルパワー × 0.5 × ランダム値 × クリティカル倍率

ランダム値 = 1.0 + Random.Range(-(防御側Luck + 1), 0) × 0.01
           = 0.99 ~ 1.0 の範囲

クリティカル判定:
  ランダム値 > -クリティカル率 の場合クリティカル
  クリティカル倍率 = 1.0 + (クリティカルダメージ × 0.1)
```

#### 4.2.2 防御計算

```
防御後ダメージ = 基礎ダメージ - (防御力 × 0.25) + 属性ダメージ

属性ダメージ = (攻撃属性値 - 相性倍率 × 防御側最大属性値) × 0.5
```

#### 4.2.3 シールド計算

```
シールドダメージ = 基礎ダメージ × シールド攻撃率 × 0.25
最終ダメージ = max(0, 防御後ダメージ - シールド値)
シールド残量 -= シールドダメージ
```

### 4.3 ガードシステム

既存実装: `CharacterControl.GuardInflicted()`, `GuardComponent.cs`

#### 4.3.1 ガードの仕様

- ガードは複数回数制（guardNum）
- 各ガードに耐久値（guardHealth）がある
- ガードが破壊されるとクールダウンタイム発生
- クールダウン後、自動で復活
- ガード中は移動速度低下（CrouchSpeedMultiplier）

#### 4.3.2 ガード破壊の流れ

```
ダメージ受ける
  ↓
guardHealth >= ダメージ ?
  ├─ YES → guardHealth減少、ダメージ無効化
  └─ NO  → ガード破壊
             ↓
        guardCooldown開始
             ↓
        アイコン非表示 + シェイクアニメーション
             ↓
        クールダウン完了
             ↓
        guardHealth = maxGuardHealth
             ↓
        アイコン復活 + パンチアニメーション
```

### 4.7 ノックバックシステム

#### 4.7.1 概要

攻撃アイテムスキルにノックバック値を設定し、ヒット時に対象を押し出したり引き寄せたりする効果を実現する。

#### 4.7.2 ノックバックパラメータ

| パラメータ | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| knockbackValue | float | 0 | ノックバック力（正: 押し出し、負: 引き寄せ） |
| knockbackLimit | float | 設定可能 | ノックバック力の上限値 |
| knockbackTarget | KnockbackTarget | Enemy | ノックバック対象（アイテムスキル依存で設定） |

**ノックバック対象（KnockbackTarget）:**
- **Enemy**: 敵のみにノックバック適用
- **Player**: プレイヤーのみにノックバック適用（自傷アイテムスキル等）
- **Both**: 敵・プレイヤー両方にノックバック適用

※ノックバック対象はアイテムスキルごとに個別設定する。

**ノックバック方向:**
- **正の値（knockbackValue > 0）**: 攻撃側と反対方向に対象を押し出す
- **負の値（knockbackValue < 0）**: 攻撃側の方向に対象を引き寄せる
- **ゼロ（knockbackValue = 0）**: ノックバック効果なし

#### 4.7.3 ノックバック計算式

```
最終ノックバック = min(knockbackValue, knockbackLimit) × (1 - 防御側軽減率)

防御側軽減率 = corePower × KNOCKBACK_RESISTANCE_MULTIPLIER
※corePowerは吹き飛ばし耐性パラメータ（BehaviourStatus）
※KNOCKBACK_RESISTANCE_MULTIPLIERはGameBalanceで定義

例:
knockbackValue = 10
knockbackLimit = 8
防御側corePower = 2.0
KNOCKBACK_RESISTANCE_MULTIPLIER = 0.1

軽減率 = 2.0 × 0.1 = 0.2 (20%軽減)
最終ノックバック = min(10, 8) × (1 - 0.2) = 8 × 0.8 = 6.4
```

#### 4.7.4 ノックバックの適用処理

```
攻撃ヒット
  ↓
knockbackValue != 0 ?
  ├─ NO  → ノックバック処理スキップ
  └─ YES → ノックバック計算開始
       ↓
上限チェック
  └─ min(knockbackValue, knockbackLimit)
       ↓
防御側の軽減計算
  └─ corePowerによる軽減適用
       ↓
方向計算
  ├─ knockbackValue > 0 → 攻撃側の反対方向
  └─ knockbackValue < 0 → 攻撃側の方向
       ↓
物理力適用
  └─ 対象にForceを加える
```

#### 4.7.5 特殊なケース

| ケース | 処理 |
|--------|------|
| 対象が壁に衝突 | ノックバック停止、追加ダメージ（オプション） |
| 対象が空中 | 通常どおり適用（落下中も有効） |
| 対象がスーパーアーマー状態 | ノックバック無効（ダメージは適用） |
| 対象がガード中 | ノックバック軽減（ガード性能による） |

#### 4.7.6 引き寄せの用途例

負の値による引き寄せ効果の活用例:
- **近接キャラクター**: 逃げる敵を引き寄せて攻撃範囲内に収める
- **コンボ継続**: 吹き飛んだ敵を引き戻して追撃
- **集敵アイテムスキル**: 複数の敵を一箇所に集める範囲アイテムスキル

#### 6.2.1 相性倍率マトリクス

```
防御側属性 →
攻撃側↓    火    水    木    雷    土    氷    油    毒    風    魂

火        1.0   0.8   1.2   0.8   1.4   0.6   0.6   1.0   0.8   0.4
水        1.2   1.0   0.6   0.4   1.0   1.4   0.0   0.8   0.8   0.8
木        0.8   1.4   1.0   0.8   1.2   0.6   0.4   0.4   0.6   1.0
雷        1.2   1.6   1.2   1.0   0.0   1.0   1.4   0.8   0.8   1.0
土        0.6   1.0   0.8   1.6   1.0   0.8   0.8   0.6   0.0   0.8
氷        1.4   0.6   1.4   1.0   1.2   1.0   0.8   0.8   1.2   1.0
油        1.4   1.6   1.6   0.6   1.2   1.2   1.0   1.2   0.8   1.0
毒        1.0   1.2   1.6   1.2   1.4   1.2   0.8   1.0   1.0   0.6
風        1.2   1.2   1.4   1.2   1.6   0.8   1.2   1.0   1.0   1.0
魂        1.6   1.2   1.0   1.0   1.2   1.0   1.0   1.4   1.0   1.0
```

**倍率の意味:**
- `1.6`: 超効果的（1.6倍ダメージ）
- `1.4`: 効果的（1.4倍ダメージ）
- `1.2`: やや効果的（1.2倍ダメージ）
- `1.0`: 普通（等倍）
- `0.8`: やや効果薄い（0.8倍ダメージ）
- `0.6`: 効果薄い（0.6倍ダメージ）
- `0.4`: ほぼ無効（0.4倍ダメージ）
- `0.0`: 完全無効（ダメージなし）

#### 6.2.2 主要な相性関係

**強力な相性 (1.6倍):**
- 雷 → 水（電気が水に伝導）
- 油 → 水（油が水面に拡散）
- 土 → 雷（土がアースとして機能）
- 魂 → 火（精神が生命力を侵食）

**完全無効 (0.0倍):**
- 水 → 油（水と油は混ざらない）
- 雷 → 土（土にアースされる）
- 土 → 風（土は風に届かない）

### 6.3 属性ダメージ計算

```
属性ダメージ = (攻撃属性値 - 相性倍率 × 防御最大属性値) × 0.5

例:
攻撃側: Electric属性 50
防御側: Aqua属性 30 (最大属性)
相性: Electric → Aqua = 1.6倍

属性ダメージ = (50 - 1.6 × 30) × 0.5
             = (50 - 48) × 0.5
             = 1.0

防御側の属性値が高いと、相性が悪くても耐えられる
```

### 8.2 レベルシステム

既存実装: `Level/LevelExp.cs`, `Level/LevelCap.cs`

#### 8.2.1 通貨消費によるレベルアップ

**重要**: 本ゲームでは戦闘で経験値を獲得せず、**通貨（ゲーム内マネー）を消費してレベルアップ**します。

```
レベルアップシステム
├── 戦闘 → 通貨獲得（アイテムドロップ売却含む）
├── 通貨消費 → レベルアップ
└── プレイヤー/パートナー両方に適用
```

**レベルアップコスト計算式（案）:**
```
必要通貨 = 基礎コスト × レベル × (1 + レベル / 100)

例: 基礎コスト = 100 の場合
Lv1→2: 100 × 1 × 1.01 = 101
Lv50→51: 100 × 50 × 1.5 = 7,500
Lv100→101: 100 × 100 × 2.0 = 20,000
Lv500→501: 100 × 500 × 6.0 = 300,000
```

#### 8.2.2 レベルアップによる成長

レベルアップ時に以下が発生:

1. **自動ステータス成長**: HP/攻撃力/防御力等が自動で上昇
2. **アビリティポイント獲得**: レベルアップ前のレベル値と同数のAPを獲得

```
例: Lv99 → Lv100 にレベルアップ
  → ステータス自動成長
  → 99 AP 獲得
```

#### 8.2.3 自動成長計算式

```
成長後ステータス = 基礎値 × (1 + (level - 1) × 成長率)

成長率 = 0.1 (10%/Lv)

例: 基礎HP 10, Lv100の場合
HP = 10 × (1 + 99 × 0.1) = 10 × 10.9 = 109
```

**Lv1000時の自動成長倍率:**
```
倍率 = 1 + 999 × 0.1 = 100.9倍
基礎HP 10 → Lv1000 HP ≈ 1009
```

### 8.5 戦闘バランス例

#### 8.5.1 設計目標

| 項目 | 目標値 |
|------|--------|
| 雑魚戦（同レベル） | 5-10回攻撃で撃破 |
| 雑魚戦時間 | 10-20秒 |
| ボス戦時間 | 3-5分 |
| 防御力軽減（同レベル） | 最大50%以上可能 |
| 属性有利ダメージ | +50% |

#### 8.5.2 序盤戦闘例（Lv10同士）

```
【プレイヤー】Lv10
HP: 19, 攻撃力: 9.5, 防御力: 3.8, アイテムスキルパワー: 9.5

【雑魚敵】Lv10
HP: 15, 攻撃力: 5.7, 防御力: 1.9

--- ダメージ計算（プレイヤー → 雑魚）---
基礎ダメージ = 9.5 × 0.5 = 4.75
防御軽減 = 1.9 × 0.25 = 0.475
最終ダメージ ≈ 4.3

撃破必要回数 = 15 ÷ 4.3 ≈ 3.5回 → 4回

--- ダメージ計算（雑魚 → プレイヤー）---
基礎ダメージ = 5.7 × 0.5 = 2.85
防御軽減 = 3.8 × 0.25 = 0.95
最終ダメージ ≈ 1.9

プレイヤー死亡まで = 19 ÷ 1.9 ≈ 10回
```

#### 8.5.3 属性有利時の戦闘例

```
属性相性 1.5倍（有利）の場合:

属性ダメージ = (攻撃属性値 - 1.5 × 防御属性値) × 0.5

例: 攻撃側属性値 10, 防御側属性値 5
属性ダメージ = (10 - 1.5 × 5) × 0.5 = (10 - 7.5) × 0.5 = 1.25

基礎ダメージ 4.75 + 属性ダメージ 1.25 = 6.0
→ 約26%のダメージ増加

※属性値を上げることでさらにダメージ増加
```

#### 8.5.4 防御50%軽減の条件

```
現在の計算式:
防御後ダメージ = 基礎ダメージ - (防御力 × 0.25)

50%軽減を達成する条件:
防御力 × 0.25 = 基礎ダメージ × 0.5
防御力 = 基礎ダメージ × 2

例: 基礎ダメージ 100 の場合
→ 防御力 200 で50%軽減

※防御特化ビルドで実現可能
```

#### 8.5.5 一撃必殺の条件（アイテムスキル特化）

```
【単体攻撃特化アイテムスキル例】
アイテムスキルパワー: 1000
クリティカル倍率: 2.0
属性有利: 1.5倍

基礎ダメージ = 1000 × 0.5 × 2.0 = 1000
属性ダメージ = +150 (仮定)
合計 = 1150

雑魚HP 15 (Lv10) に対して
→ 即死（ダメージ過剰）

※高レベルアイテムスキル + 攻撃特化ビルドで実現
```

【選択肢】
A. ステータス重視 → 基礎能力が高く安定
B. アビリティ重視 → 多彩なアビリティで戦術幅が広い
C. バランス型 → 汎用性重視

※リセット機能の有無は要検討
```

### 8.7 キャラクター成長システム

#### 8.7.1 成長の二軸構造（ポケモン型）

キャラクターの成長は「種族値」と「努力値」の二軸で構成される：

| 概念 | ゲーム内名称 | 説明 |
|-----|------------|------|
| 種族値 | 固有成長曲線 | キャラクター固有の成長率、変更不可 |
| 努力値 | AP振り分け | プレイヤーが自由に配分可能 |

**種族値（固有成長曲線）**:
- 各キャラクターはレベルアップ時の自動成長率が異なる
- 例：火属性キャラ → HP成長率が高い、雷属性キャラ → 速度成長率が高い
- 同一キャラでも個体差あり（入手時にランダム決定）

**努力値（AP振り分け）**:
- レベルアップで獲得するAPをプレイヤーが自由に配分
- 振り分け先：ステータス強化、アビリティツリー解放
- リセット機能あり（通貨消費）

#### 8.7.2 推奨ビルドシステム

アビリティツリー画面に「おすすめ」ボタンを配置：

| 機能 | 説明 |
|-----|------|
| おすすめボタン | 押すと推奨ビルドを自動適用 |
| 最適化基準 | 戦闘力（総合的な強さ指標）を最大化 |
| ビルド提案 | キャラクターの種族値を考慮した最適配分 |

**推奨ビルドの種類**:
```
【自動提案される主要ビルド】
1. バランス型 - 汎用性重視、初心者向け
2. 攻撃特化型 - DPS最大化
3. 防御特化型 - タンク向け
4. 属性特化型 - キャラ固有属性を活かす
```

**設計方針**:
- 初心者は「おすすめ」ボタンで迷わず成長可能
- 上級者は手動で細かくカスタマイズ可能
- 「試行錯誤の楽しさ」と「手軽さ」を両立

#### 8.7.3 戦闘力計算

キャラクターの総合的な強さを示す指標：

```
戦闘力 = Σ(各ステータス × 重み係数)

重み係数例:
- HP: 1.0
- 攻撃力: 1.5
- 防御力: 1.2
- 属性値: 1.0
- アイテムスキルパワー: 1.3
```

**用途**:
- 推奨ビルドの最適化基準
- ステージ推奨戦闘力の表示
- プレイヤー間の比較指標

## 9. シナジーシステム

### 9.1 設計方針
**他ゲーム以上のシナジー数を実装**することを目標とし、アイテムスキルと装備の組み合わせによる「ビルド構築」の楽しさを重視する。

### 9.2 アイテムスキルシナジー

#### 9.2.1 属性連携
特定の属性アイテムスキルを**連続ヒットさせる**ことでボーナス効果が発生：

**発動条件**: 先行アイテムスキルが命中した後、一定時間内に後続アイテムスキルを命中させる
- 例: 油アイテムスキル命中 → 火アイテムスキル命中 で「追加ダメージ＋範囲拡大」発動

| 先行アイテムスキル | 後続アイテムスキル | 効果 |
|-----------|-----------|------|
| 油 | 火 | 追加ダメージ＋範囲拡大 |
| 水 | 雷 | ダメージ伝播＋感電 |
| 水 | 氷 | 凍結＋移動速度低下 |
| 毒 | 木 | 昏睡中に木ダメージ増加 |

#### 9.2.2 アイテムスキルコンボ
特定のアイテムスキルを連続使用することでコンボボーナス：

- **2連携**: ダメージ +10%
- **3連携**: ダメージ +25%、追加効果発動
- **4連携以上**: ダメージ +50%、特殊演出

### 9.3 装備セット効果

#### 9.3.1 セット構成
同シリーズ装備を揃えることでセットボーナスが発動：

| セット数 | ボーナス |
|---------|---------|
| 2個 | 軽いステータスボーナス |
| 3個 | 中程度ボーナス＋パッシブ効果 |
| 4個（完全セット） | 大幅なプレイスタイル変化 |

#### 9.3.2 セット例（案）
- **炎獄セット**: 継続ダメージ特化、DoT効果増加
- **氷結セット**: クリティカル特化、凍結時追加ダメージ
- **疾風セット**: 速度特化、回避時反撃
- **大地セット**: 防御特化、ガード時HP回復

### 9.4 シナジー発見システム
- **未発見シナジー**: 初回発見時に報酬
- **シナジー図鑑**: 発見したシナジーを記録
- **ヒントシステム**: 一定条件で未発見シナジーのヒント表示

## 10. ステージ難易度システム

### 10.1 概要

ステージには3段階の難易度が存在し、難易度に応じて死亡時のペナルティと敵からのドロップ報酬が変化する。
難易度はステージ選択時にのみ変更可能（ステージ攻略中は変更不可）。

### 10.2 難易度レベル

| 難易度 | 英名 | 敵レベル補正 | 報酬倍率 | 対象プレイヤー |
|--------|------|-------------|---------|---------------|
| ノーマル | Normal | +0 | 1.0x | 初心者〜中級者 |
| ハード | Hard | +10 | 1.5x | 中級者〜上級者 |
| ウルトラ | Ultra | +20 | 2.5x | 上級者・やり込み |

### 10.3 死亡ペナルティ

#### 10.3.1 ノーマル難易度

| 項目 | ペナルティ           |
|------|-----------------|
| 所持金 | 10％ロスト          |
| 装備品 | ペナルティなし         |
| キャラクター | ペナルティなし         |
| 進行状況 | ステージ開始地点からリスタート |

#### 10.3.2 ハード難易度

| 項目 | ペナルティ |
|------|-----------|
| 所持金 | 50%ロスト |
| 装備品 | 一部ドロップ（ステージ内で回収可能） |
| キャラクター | ペナルティなし |
| 進行状況 | ステージ開始地点からリスタート |

**装備品ドロップの仕様:**
- 死亡時に装備中のアイテムの一部がその場にドロップ
- ドロップ確率: アイテムのレアリティに依存
- 回収制限時間: 10分（GameBalanceで調整可能）
- 制限時間内にステージに再挑戦し、ドロップ地点まで到達すれば回収可能
- 制限時間を過ぎた場合、アイテムは消滅

#### 10.3.3 ウルトラ難易度

| 項目 | ペナルティ |
|------|-----------|
| 所持金 | 100%ロスト（ステージ内で回収可能） |
| 装備品 | 全てドロップ（ステージ内で回収可能） |
| キャラクター | ドロップ（ステージ内で回収可能） |
| 進行状況 | ステージ開始地点からリスタート |

**キャラクタードロップの仕様:**
- 死亡したキャラクターはその場にドロップし、パーティから一時的に離脱
- 別のキャラクターでステージに再挑戦し、ドロップ地点まで到達すれば復帰可能
- 回収制限時間: 30分（GameBalanceで調整可能）
- 制限時間を過ぎた場合、キャラクターは永久ロスト

### 10.4 敵ドロップシステム

#### 10.4.1 ドロップ報酬倍率

| 難易度 | 通貨ドロップ | アイテムドロップ率 | レアアイテム率 |
|--------|------------|------------------|---------------|
| ノーマル | 1.0x | 1.0x | 1.0x |
| ハード | 1.5x | 1.3x | 1.5x |
| ウルトラ | 2.5x | 2.0x | 3.0x |

#### 10.4.2 敵レベル補正

難易度に応じて敵のレベルが補正される:

```
敵の実効レベル = 基礎レベル + 難易度補正

例: 基礎レベル30の敵
ノーマル: 30 + 0 = Lv30
ハード: 30 + 10 = Lv40
ウルトラ: 30 + 20 = Lv50
```

**レベル補正の影響:**
- 敵のHP、攻撃力、防御力が上昇
- ドロップアイテムの品質が向上
- 経験値（通貨換算）が増加

### 10.5 ドロップアイテム回収システム

#### 10.5.1 回収可能アイテム

| アイテム種別 | 回収制限時間 | 回収失敗時 |
|------------|------------|-----------|
| 通貨 | 10分 | 消滅 |
| 装備品 | 10分 | 消滅 |
| キャラクター | 30分 | 永久ロスト |

#### 10.5.2 回収の流れ

```
プレイヤー死亡
  ↓
アイテム/キャラクターがその場にドロップ
  ↓
回収タイマー開始
  ↓
別キャラクターで再挑戦
  ↓
ドロップ地点に到達
  ├─ 到達成功 → アイテム/キャラクター回収
  └─ タイムアウト → アイテム消滅/キャラクターロスト
```

#### 10.5.3 回収マーカー

- ドロップ地点はマップ上にマーカーで表示
- マーカーには残り時間が表示される
- キャラクタードロップの場合は特別なアイコンで表示

### 10.6 難易度選択UI

#### 10.6.1 ステージ選択画面

```
┌─────────────────────────────────────┐
│  ステージ名: 炎の洞窟               │
│  推奨戦闘力: 5,000                  │
├─────────────────────────────────────┤
│  難易度選択:                        │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐│
│  │ノーマル │ │ ハード  │ │ウルトラ ││
│  │敵Lv +0  │ │敵Lv +10 │ │敵Lv +20 ││
│  │報酬 x1  │ │報酬 x1.5│ │報酬 x2.5││
│  └─────────┘ └─────────┘ └─────────┘│
├─────────────────────────────────────┤
│  死亡ペナルティ: [選択難易度に応じて表示]│
│                                     │
│          [挑戦する]                 │

#### 10.6.2 難易度変更の制限

- ステージ選択時のみ難易度変更可能
- ステージ攻略中は難易度変更不可
- 一度ステージに入ると、クリアまたはリタイアまで同じ難易度
