## 1. キャラクター収集・選択システム

### 1.0 基本概念

**このゲームに固定の主人公キャラクターは存在しない。**
プレイヤーはゲームプレイを通じてキャラクターを収集し、「保有コレクション」の中から任意のキャラクターを「操作キャラクター」として選択してプレイする。

```
保有コレクション（OwnedCharacterCollection）
  ├── 操作キャラクター（1体）  ← プレイヤーが直接操作
  ├── パートナー（最大2体）   ← AIが自動操作
  └── 待機キャラクター       ← 出撃していないキャラクター
```

#### 1.0.1 OwnedCharacterCollection データ構造

```csharp
/// <summary>
/// プレイヤーが保有するキャラクター全体を管理するコレクション
/// </summary>
public class OwnedCharacterCollection
{
    public List<OwnedCharacterData> characters;  // 保有キャラクター一覧
    public int maxCapacity = 50;                 // 基本登録上限

    public OwnedCharacterData operatingCharacter; // 現在操作中のキャラクター
    public List<OwnedCharacterData> partners;     // パートナー（最大2体分）
}

/// <summary>
/// 保有キャラクター1体分のデータ
/// プレイヤー操作・AI操作いずれにも使用する共通データ構造
/// キャラクターはアイテムの一区分であり、アイテムレアリティ体系を共有する
/// </summary>
public class OwnedCharacterData
{
    public int characterId;          // キャラクター種別ID（敵マスタと共通）
    public string uniqueId;          // 保有インスタンスのユニークID（同一キャラの複数体を区別）
    public int currentLevel;         // 現在のレベル（入手時はLv1から）
    public int originalLevel;        // 敵として登場した時のレベル
    public Rarity rarity;            // レアリティ（アイテムと同一体系・レベルキャップを決定）
    public bool isAlchemizable;      // 錬金術可否（特別なキャラクターは false）
    public StatusData baseStatus;    // 基礎ステータス
    public StatusData currentStatus; // 現在ステータス（装備込み）
    public CharacterSlotItemData equipment; // 装備スロット
    public PartnerSize size;         // 大型/小型（出撃枠消費に影響）
    public PartnerBehavior behavior; // AI行動設定
    public bool isActive;            // 出撃中かどうか
    public bool isDeadInStage;       // ステージ中の死亡状態（ステージ終了でリセット）
}
```

#### 1.0.2 操作キャラクター切り替えフロー

**出撃画面（ステージ開始前）**
```
出撃画面
  ↓
保有コレクション一覧表示
  ↓
操作キャラクター選択（1体）
  + パートナー選択（最大2スロット: Small=0.5, Normal=1.0）
  + 各キャラクターの装備設定
  ↓
OwnedCharacterCollection に選択を記録
  ↓
ステージ開始 → 選択した操作キャラクターで CharacterControl を初期化
```

**ステージ中（手動切り替え）**
```
パーティUI（名前＋ステータス）常時表示
  ↓
生存キャラクターのUI をタップ
  ↓
PlayerCharacterController.SwitchOperator(target)
  ├─ 元の操作キャラクター → AI操作モードへ
  └─ 対象キャラクター → プレイヤー操作モードへ
※ コスト・クールダウンなし / 死亡キャラはタップ不可
```

**ステージ中（死亡時の自動引き継ぎ）**
```
操作キャラクター死亡
  ↓
isDeadInStage = true
  ↓
生存パートナーが存在する？
  ├─ Yes → リスト順で次の生存キャラクターを自動選択してSwitchOperator()
  └─ No  → 全員死亡 → ゲームオーバー処理
```

**死亡・リスポーン**

| 状況 | 挙動 |
|------|------|
| 操作キャラ死亡 | 次の生存キャラへ自動移譲 |
| 全員死亡（オンライン） | ゲームオーバー → 課金リスポーン（全員全回復）or 拠点送還（難易度別ペナルティ） |
| 全員死亡（オフライン） | チェックポイントから再開。全員HP1/4・その他初期状態で復活 |

### 1.1 キャラクター入手システム

#### 1.1.0 初期キャラクター選択（ゲーム開始時）

ゲーム開始直後は保有コレクションが0体のため、ストーリー上「たまたま落ちていたキャラクターアイテム」を拾うシーンで初期キャラクターを選択する。

```
チュートリアル演出
  ↓
5〜10体の初期候補を提示（属性ごとに1〜2体）
  ↓
プレイヤーが1体を選択 → 操作キャラクターとして登録
  ↓
選ばなかったキャラクターはゲーム内で通常入手可能
```

- 各初期候補キャラクターはステータスが明確に異なる（序盤の難易度に影響）
- 選択肢の体数・属性構成は今後確定（現状: 属性5〜10種×1〜2体）

#### 1.1.1 入手条件
- **入手タイミング**: ボス撃破時のみ
- **ドロップ判定**: オーバーキル具合に応じた確率

#### 1.1.2 ドロップ率計算

```
基本ドロップ率 = キャラ固有確率 × オーバーキル補正

オーバーキル補正:
  HP0ちょうどで撃破 → 50%
  HP × 0.5 のダメージ → 80%
  HP × 2.0 のダメージ → 10%

最終確率 = min(基本ドロップ率, 上限確率)
```

#### 1.1.3 同一キャラ入手時の選択

```
同一キャラ入手
  ↓
選択UI表示
  ├─ 選択A: 既存キャラの基礎ステータスをアップ
  │         └─ 各ステータス +2-5% (レアリティ依存)
  └─ 選択B: 追加で登録
             └─ 登録枠を1消費
```

### 1.2 パートナーシステム

#### 1.2.1 パートナー制限
- **牽引可能数**: 2体まで（小型は2体で1体分）
- **登録上限**: 基本50体（サブスクで拡張可能）

#### 1.2.2 キャラクターデータの統一構造
**重要**: プレイヤーキャラクターとパートナーは**同一のデータ構造**を使用する。

- すべての保有キャラクターは同じ `CharacterData` 構造で管理
- 「プレイヤー」と「パートナー」の区別は出撃時に決定
- どのキャラクターもプレイヤーとして操作可能

#### 1.2.3 出撃画面システム

```
出撃画面
├── 保有キャラクター一覧表示
├── プレイヤーキャラクター選択（1体）
│   └── 選択したキャラクターを直接操作
├── パートナーキャラクター選択（最大2体）
│   └── AIが自動操作
└── 出撃確認
```

#### 1.2.4 パートナーの特性

| 項目 | 説明 |
|------|------|
| レベル | 仲間化時はLv1からスタート |
| レベルアップ | **通貨を消費**してレベルアップ（プレイヤーと同じ方式） |
| ステータス | 元のレベルまで上げると敵対時と同じステータス |
| 装備 | プレイヤーと装備を兼用可能 |
| AP振り分け | プレイヤーと同様にアビリティポイント振り分け可能 |
| アビリティツリー | プレイヤーと同様にアビリティツリー選択可能 |
| AI | 自動で敵を攻撃（設定で調整可能） |

### 1.3 コレクション管理メニュー

拠点でいつでも実施できる3種の操作。コレクション満員時は帰還後に強制実行。

#### 1.3.1 おわかれ

キャラクターを解放し、そのキャラクター種別に固定されたアイテム候補からランダムにアイテムを受け取る。

```csharp
// キャラクター種別ごとにドロップ候補テーブルを定義
// (CharacterMasterData.farewellItemPool)
Item[] dropped = RandomPickFromPool(character.farewellItemPool);
collection.Remove(character);
```

#### 1.3.2 スクラップ

キャラクターを解体して通貨を回収する。

```
返還通貨 = そのキャラクターのレベルアップに費やした総通貨 × 0.5
```

#### 1.3.3 錬金術（alchemy）

同一キャラクターを素材に合成して、ベースキャラクターのレアリティを1段階上げる。

**条件:**
- ベース・素材ともに**同一キャラクター種別**（characterId が一致）
- 素材のレアリティ ≥ ベースのレアリティ
- ベースの `isAlchemizable == true`

**必要素材数（レアリティ別）:**

| ベースのレアリティ | 必要素材数 |
|------------------|-----------|
| コモン | 1体 |
| アンコモン | 2体 |
| レア | 3体 |
| ～以降 | 要バランス調整（`Status - Rarelity.csv` 参照） |

**結果:**
- ベースキャラクターのレアリティが1段階上昇（レベルキャップも上昇）
- ベースのレベル・APビルド・uniqueId はそのまま保持
- 素材キャラクター: 装備品を全て返却 + レベルコスト × 0.5 の通貨を返還してから削除

```csharp
// 錬金術フロー
bool Alchemize(OwnedCharacterData baseChar, List<OwnedCharacterData> materials)
{
    if (!baseChar.isAlchemizable) return false;
    if (!materials.All(m => m.characterId == baseChar.characterId
                         && m.rarity >= baseChar.rarity)) return false;
    if (materials.Count < GetRequiredCount(baseChar.rarity)) return false;

    foreach (var m in materials)
    {
        ReturnEquipment(m);
        RefundCurrency(m, 0.5f);
        collection.Remove(m);
    }
    baseChar.rarity++;
    return true;
}
```

#### 1.3.4 コレクション満員時の超過取得フロー

```
ステージ中にボスがキャラをドロップ（コレクション満員）
  ↓
「保留キャラクター」としてセッション中に一時保持
  ↓
ゲームオーバーになった場合 → 保留キャラ消滅
  ↓
拠点に帰還
  ↓
削除選択UI（強制）
  ├─ 保留キャラを削除（おわかれ/スクラップ）
  └─ 既存キャラを削除（おわかれ/スクラップ/錬金術）して保留キャラを登録
```

### 1.4 登録上限と拡張

```
基本登録枠: 50体
  ↓
枠が満杯
  ├─ コレクション管理メニュー（1.3）で1体を削除
```

## 2. 敵AIシステム

### 2.1 AI状態機械

#### 2.1.1 基本AI状態

```
待機 (Idle)
  ↓ プレイヤー検知
追跡 (Chase)
  ↓ 射程内
攻撃 (Attack)
  ↓ HP低下
逃走 (Flee)
  ↓ 体勢立て直し
待機に戻る
```

#### 2.1.2 状態遷移パラメータ

| パラメータ | 説明 | デフォルト値 |
|-----------|------|-------------|
| detectionRange | プレイヤー検知範囲 | 10.0 |
| loseTargetRange | 追跡解除範囲 | 15.0 |
| attackRange | 攻撃開始距離 | 2.0 (近接) / 8.0 (遠隔) |
| fleeThreshold | 逃走開始HP割合 | 0.2 (20%) |
| reengageThreshold | 再攻撃開始HP割合 | 0.3 (30%) |
| idleWaitTime | 待機時間（秒） | 1.0-3.0 |
| attackCooldown | 攻撃後の待機時間 | 0.5-2.0 |

#### 2.1.3 状態遷移図（詳細）

```
                    ┌─────────────────────────────────┐
                    │                                 │
                    ▼                                 │

              │   Idle   │                           │
```

```
  ↓
インベントリ表示
  ├─ 各アイテムに「保護」ボタン
  └─ 保護コスト表示
  ↓
保護したいアイテムを選択
  ↓
通貨消費して保護状態に
  └─ 保護アイテムには🛡️マーク表示
  ↓
出撃
```

### 5.3 ペナルティ軽減（課金）

| 方法 | 効果 |
|------|------|
| 即リスポーンアイテム（課金） | その場で復活、ペナルティなし |
| 切断時無敵アイテム（課金） | 通信切断時のキャラクター保護 |
| パートナーの救援 | パートナーがHP回復（条件付き） |

### 5.4 死亡処理フロー

```
プレイヤーHP <= 0
  ↓
死亡演出
  ↓
即リスポーンアイテム確認
  ├─ 所持あり → 使用確認UI → 使用 → その場で復活
  └─ 所持なし or 使用しない
       ↓
アイテムロスト処理
  ├─ 保護されていないアイテムを失う
  └─ 保護アイテムは維持
       ↓
リスポーン画面
  └─ 最後のチェックポイントから再開
```

## 7. 環境インタラクションシステム

### 7.1 属性と環境の連動

属性攻撃がフィールド上の要素と相互作用するシステム。

#### 7.1.1 基本インタラクション（初期実装）

| 環境要素 | 属性 | 効果 |
|---------|------|------|
| 水場 | 雷 | 水面に電気拡散、範囲内の敵にダメージ |
| 油溜まり | 火 | 引火して範囲継続ダメージ |
| 草むら | 火 | 燃焼、視界遮蔽物の除去 |
| 氷面 | 火 | 溶けて水場に変化 |
| 水場 | 氷 | 凍結して歩行可能に |

#### 7.1.2 連鎖反応

```
例: 油 → 火 → 水
  ↓
油アイテムスキルで油溜まり生成
  ↓
火アイテムスキルで引火（範囲ダメージ）
  ↓
水アイテムスキルで消火（蒸気発生、視界遮蔽）
```

### 7.2 将来実装予定（物理演算）

#### 7.2.1 地形変化

| アクション | 効果 |
|-----------|------|
| 爆発 | 地形破壊、新ルート開通 |
| 土属性アイテムスキル | 壁生成、足場作成 |
| 大質量攻撃 | 床崩落 |

#### 7.2.2 液体シミュレーション

| 要素 | 挙動 |
|------|------|
| 水 | 低所に流れる、溜まる |
| 油 | 水面に浮く、広がる |
| 毒液 | 継続ダメージエリア化 |

### 7.3 インタラクション設計方針

- **発見の楽しさ**: 組み合わせの発見を報酬とする
- **バランス**: 強力すぎる組み合わせも「発見」として許容
- **段階的実装**: 基本→高度な物理演算へ
